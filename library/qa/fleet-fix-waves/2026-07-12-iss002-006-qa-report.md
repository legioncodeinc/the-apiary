## Warnings (should fix)

### W-1 â€” Search-hit type badge dead on the wire: daemon sends `memoryType`, hive reads `type`

- **Where:**
  - honeycomb `src/daemon/runtime/memories/recall.ts:824-825` â€” fused memories-source hits are stamped `{ memoryId: doc.id }` and `{ memoryType: doc.memoryType }`; hits are forwarded VERBATIM by `recallResponse` (`src/daemon/runtime/memories/api.ts:527`), so the on-wire key is `memoryType`.
  - hive `src/dashboard/web/wire.ts:529-531` â€” `RecallHitSchema` adds `memoryId` (matches) and `type: z.string().optional()` (does NOT match; no `memoryType` field), then maps `type: h.type ?? ""` (`wire.ts:2582`).
  - hive `src/dashboard/web/pages/memories.tsx:188-204` â€” `searchHitRecord` copies `hit.type` into the record; `MemoryRow` renders `record.type || "fact"`.
- **Effect:** honeycomb's presentation-parity projection of the taxonomy tag (threaded deliberately through the lexical arm, the `<#>` semantic arm, the hydrate SELECT, the local ANN index, and RRF fusion â€” `recall.ts` `typeColumn`, `local-vector-index.ts` `memoryType`) is never consumed. Every memory found via search shows the `fact` badge even when it is a `decision`/`preference`/etc., while the same memory in the browse list shows its true type. Actionability (click/edit/forget via `memoryId`) is unaffected â€” the primary criterion still holds.
- **Why it slipped:** each repo pinned its own half â€” honeycomb's parity suite asserts `hit.memoryType` (`tests/daemon/runtime/memories/iss-006-search-list-corpus-parity.test.ts:496`), hive's vintage-matrix fixtures hand-author hits with `type` (`tests/dashboard/search-result-card-parity.test.tsx:361-372`). No cross-repo contract fixture uses a real daemon payload with a non-default type.
- **Recommended remediation (hive, one line + one test):** widen the schema/mapping to `memoryType ?? type ?? ""` (add `memoryType: z.string().optional().catch(undefined)` to `RecallHitSchema`, prefer it in the `type` mapping), and add a fixture whose hit carries `memoryType: "decision"` asserting the rendered badge. Alternatively honeycomb could additionally alias `type` on the hit, but the hive-side read is the smaller, back-compatible change.
