Install And Uninstall

Shared memory for your AI coding agents — one command, all local.

This page exists so you can read the installer before you run it. Nothing here pipes anything automatically — copy a line when you're ready.

1Install

macOS & Linux:

# macOS / Linux curl -fsSL {{VANITY}} | sh

Windows (PowerShell):

# Windows PowerShell irm {{VANITY}}/install.ps1 | iex

2Uninstall

macOS & Linux:

# macOS / Linux curl -fsSL {{VANITY}}/uninstall | sh

Windows (PowerShell):

# Windows PowerShell irm {{VANITY}}/uninstall.ps1 | iex

3Verify before you run

Prefer to check the bytes first? Download the script and its checksum, verify, then run:

# download + verify + run (macOS / Linux) curl -fsSL -O {{VANITY}}/install.sh curl -fsSL -O {{VANITY}}/uninstall.sh curl -fsSL -O {{VANITY}}/SHA256SUMS sha256sum -c --ignore-missing SHA256SUMS # expect: install.sh: OK, uninstall.sh: OK sh install.sh

On macOS without sha256sum, use shasum -a 256 -c --ignore-missing SHA256SUMS. (--ignore-missing skips the checksum entries for scripts you did not download, such as the PowerShell twins.)

5What install actually does

  • Installs Node (via fnm, under your home dir, no admin) — but only if you don't already have it.
  • Installs the npm package @legioncodeinc/honeycomb globally.
  • Starts the local daemon and waits until it's healthy.
  • Opens the dashboard in your browser at a loopback address.
Everything runs locally on your machine — the daemon and dashboard bind to loopback. The installer is idempotent: re-running it is safe and skips anything already in place.
Advanced: choosing which products to install

By default this installs Honeycomb + the Doctor watchdog. Pass --products= to install more of the fleet, e.g. curl -fsSL {{VANITY}} | sh -s -- --products=honeycomb,hive,nectar. Other flags: --profile=full (a named preset), --code=<code> (a product code resolving to a preset), --dry-run (preview only, changes nothing), and --no-doctor. Environment variables (HONEYCOMB_INSTALL_PRODUCTS, HONEYCOMB_INSTALL_PROFILE, HONEYCOMB_INSTALL_LICENSE, HONEYCOMB_INSTALL_CODE) and a ~/.honeycomb/install.conf file work the same way, for administrators pinning a fleet-wide deploy shape. See the source below for the full, documented precedence.
