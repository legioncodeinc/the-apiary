# RFC 9000 - QUIC: A UDP-Based Multiplexed and Secure Transport (May 2021)

## Summary
RFC 9000 defines QUIC version 1, the transport protocol underlying HTTP/3. QUIC runs over UDP and integrates TLS 1.3 into the handshake, eliminating the separate TLS handshake round-trips of TCP+TLS. Key features: independent streams (no HOL blocking), connection migration (connections survive IP/port changes via connection IDs), 0-RTT reconnection for returning clients, built-in congestion control. QUIC uses connection IDs instead of 4-tuples for routing, enabling migration. UDP/443 must be open for QUIC; firewalls or ISPs blocking UDP port 443 silently force fallback to HTTP/2 over TCP.

## Key quotations / statistics
- "QUIC is a new multiplexed transport over UDP." (Abstract)
- "QUIC eliminates head-of-line blocking within a connection by using a different stream abstraction than TCP." (§1)
- "Connection migration is a QUIC feature that allows a client to change its network address while preserving an open connection." (§9)
- 0-RTT: "Using 0-RTT is optional. Not all implementations will choose to enable 0-RTT." (§7.4). 0-RTT carries replay attack risk.
- "QUIC requires TLS 1.3 or higher. Earlier versions of TLS MUST NOT be used." (§4)
