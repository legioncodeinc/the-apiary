# RFC 9114 - HTTP/3 (June 2022)

## Summary
RFC 9114 defines HTTP/3, the mapping of HTTP semantics onto QUIC (RFC 9000). HTTP/3 replaces TCP+TLS with QUIC as the transport layer, eliminating TCP head-of-line blocking through independent QUIC stream multiplexing. HTTP/3 uses QPACK (RFC 9204) for header compression instead of HPACK. The protocol is identified by ALPN h3. Key differences from HTTP/2: each request/response uses a separate QUIC stream (no head-of-line blocking even at transport layer); QUIC has built-in TLS 1.3; connection migration allows changing IP/port mid-session. Server Push exists in RFC 9114 but was deprecated in Chrome and removed from Safari; it is now considered harmful in practice.

## Key quotations / statistics
- "HTTP/3 relies on QUIC version 1 [QUIC-TRANSPORT], which provides streams as its basic service abstraction." (§2)
- "HTTP/3 uses QPACK [QPACK] for header field compression... QPACK is defined in a separate specification to allow for independent evolution." (§4.3)
- "Unlike HTTP/2, HTTP/3 does not define a 'priority' signal... HTTP/3 implementations SHOULD support the priority signaling scheme described in [HTTP-PRIORITY]."
- Alt-Svc (RFC 7838) or DNS HTTPS records are the discovery mechanisms for h3.
- 0-RTT: "HTTP/3 can use 0-RTT, but 0-RTT is subject to replay attacks." §10.4 — applications must not use 0-RTT for non-idempotent requests without replay protection.
