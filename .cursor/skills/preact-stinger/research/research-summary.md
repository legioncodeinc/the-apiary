## 5 Most influential sources

### 1. `external/2026-05-20-signals-api-v2-guide.md`
**Why it matters:** Confirms that `@preact/signals` v2 (latest: 2.9.0, Mar 2026) introduced a substantial new API surface - `createModel`, `useModel`, `action`, plus the `Show`/`For` utility components and `useLiveSignal`/`useSignalRef` hooks. The command brief's open question ("does v2 change the API enough to warrant a separate guide section?") is answered: YES. `guides/01-signals-api.md` must cover both the classic primitives AND the model pattern.

### 2. `external/2026-05-20-preact11-breaking-changes.md`
**Why it matters:** Preact 11 beta shipped Aug 2025; the tracking PR merged Jun 2025. The breaking changes are moderate but several directly affect the compat layer: `defaultProps` and automatic `px` suffixing moved to `preact/compat`, refs forward by default, `module.js` export renamed to `.mjs`. This shapes the migration checklist in `guides/02-compat-migration.md`.

### 3. `external/2026-05-20-preact-compat-compatibility.md`
**Why it matters:** Confirms the two most critical compat gaps: (1) React 19 `use()` hook is NOT supported, (2) `preact/compat` is incompatible with `@types/react` - never use both together, and (3) RSC + Next.js App Router is BLOCKED/footgun. These are the three things that will cause a migration to silently or noisily fail.

### 4. `external/2026-05-20-astro-preact-integration.md`
**Why it matters:** `@astrojs/preact` 5.1.2 (Apr 2026) is a major version bump. The critical find: the `useId` collision bug across Astro islands was fixed in 5.0.1 (Mar 2026). This is a correctness bug affecting accessibility that stinger-forge must document as a "require >= 5.0.1" constraint. The 5-directive table (`client:load`, `client:idle`, `client:visible`, `client:media`, `client:only`) is the core of `guides/04-astro-integration.md`.

### 5. `external/2026-05-20-fresh-framework-islands.md`
**Why it matters:** Fresh 2.2.2 (Mar 2026) uses Vite now (Fresh 1.x used esbuild). The serializable props constraint for islands is the #1 gotcha: functions and class instances CANNOT be passed as island props. Preact Signals CAN be passed (and sharing signals between islands is a powerful Fresh-specific pattern). The Fresh Vite plugin automatically aliases React to Preact, enabling npm packages that depend on React to work out of the box.

## Open questions for stinger-forge

1. **Preact 11 stable release date:** Research shows beta (11.0.0-beta.0, Aug 2025). Is v11 stable by the time stinger-forge runs? Check https://github.com/preactjs/preact/releases. If stable, update version anchors in SKILL.md accordingly; if still beta, document the dual-track (v10 stable vs v11 beta).

2. **`@preact/signals` v2 breaking vs additive:** Research shows v2 is a major version jump (from 1.x) but the core API is unchanged. Verify whether existing `signal`/`computed`/`effect` code requires migration when upgrading from v1 to v2 - it appears additive, but stinger-forge should confirm by reading the v2.0.0 changelog.

3. **Fresh 2.x vs 1.x migration path:** The docs at `fresh.deno.dev` serve Fresh 2; the 1.x docs are at a `/docs/1.x/` path. Verify whether existing Fresh 1.x projects need a migration guide (likely yes - Fresh 2 is Vite-based, Fresh 1 was esbuild-based). This may warrant a migration section in `guides/05-fresh-framework.md`.

4. **`preact/compat` + React 18 hooks (`useTransition`, `useSyncExternalStore`):** Research confirmed `useTransition` is NOT supported (no Concurrent Mode / fiber). Research did not find a definitive list of React 18 hooks and their compat status. stinger-forge should fetch the compat source or changelog to produce a complete compatibility table.

5. **`@preact/signals-react` vs `@preact/signals`:** Both exist. Research found `@preact/signals-react` 3.10.1 (May 11, 2026) - this is the React adapter, not the Preact adapter. The command brief focuses on Preact use, but if an existing React app is being progressively migrated (using signals before full Preact switch), this package bridges the gap. stinger-forge should acknowledge this path exists in `guides/00-when-to-choose-preact.md`.
