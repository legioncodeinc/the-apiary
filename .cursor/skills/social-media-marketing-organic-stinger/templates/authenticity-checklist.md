# Authenticity Checklist (Portable)

Use before publishing any post. Full rationale for each item in `guides/05-authenticity-checklist.md`.

---

```
Post review: [Platform] | [Date] | [Post type: insight/build-update/engagement]

[ ] 1. Not AI-writeable without the specific details present
[ ] 2. Contains at least one fabrication-proof specific
        (name/role, date, number, concrete situation)
[ ] 3. Sounds like the founder's actual voice (vs. "brand voice")
[ ] 4. Natural contractions present (4+ per 200 words)
[ ] 5. Does NOT start with "Excited to..." or "Thrilled to..."
[ ] 6. Fewer than 3 em-dashes in the entire post
[ ] 7. One specific CTA if any (not "like and share")
[ ] 8. Customer-embarrassment test passed
[ ] 9. Will generate comments, not just passive likes
[ ] 10. Platform-native format and register
[ ] 11. 70/30 content ratio maintained for this account
[ ] 12. Every claim defensible in a DM
```

**Red flags that require revision before posting:**
- Fails items 1, 2, or 3 → rewrite from scratch using voice anchors
- Fails item 5 → change the opening sentence
- Fails item 6 → replace em-dashes with commas or periods
- Fails item 9 → add a genuine question, counterintuitive statement, or specific controversy
