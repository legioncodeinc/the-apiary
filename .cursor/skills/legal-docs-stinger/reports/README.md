# Reports

Dated audit reports from legal document reviews accumulate here. Each report corresponds to a specific review cycle.

## Naming convention

```
YYYY-MM-DD-<company>-<document-type>-audit.md
```

Examples:
- `2026-05-20-acme-corp-dpa-audit.md` — DPA review for Acme Corp
- `2026-06-01-privacy-policy-annual-review.md` — annual privacy policy review

## When to create a report

- After completing a DPA triage using `guides/07-customer-dpa-workflow.md`
- After auditing an existing legal document against the section checklists in `guides/01` through `guides/05`
- After a regulation-change-triggered review (CCPA amendment, new EDPB guidance, etc.)

## Report structure

Each report should include:
1. Executive summary (document reviewed, date, reviewer, outcome)
2. Section-by-section findings (using the relevant guide's checklist)
3. Gaps identified (missing clauses, outdated language, regime misalignment)
4. Recommended actions (prioritized by risk level)
5. Attorney review status and sign-off

## Retention

Reports should be retained for 5 years (GDPR audit period). Do not delete reports even after a document is updated.
