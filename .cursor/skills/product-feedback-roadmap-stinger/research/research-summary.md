## Key finding per guide area

| Guide | Key finding |
|-------|------------|
| Platform selection | Featurebase is pivoting toward live chat (2026); this is a significant risk for teams choosing it as their primary feedback tool. Canny's AI features are included at no extra cost. Productlane is the clear Linear-first choice. |
| Collection surface | Voting boards produce fundamentally different signal than support tickets: votes are tracked per user with MRR/plan data, enabling RICE Reach weighting by customer value. |
| De-duplication | Semantic tagging with a 10-category ceiling is the minimal viable de-dup system. AI duplicate detection (Canny Pro, Featurebase) helps as a first pass but requires human confirmation. |
| Status transitions | Userback's "public status" feature (mapping many internal statuses to one public label) is a key UX detail that keeps internal workflows flexible without confusing customers. |
| Prioritization | RICE confidence scale: 50% = qualitative, 80% = user research, 100% = strong prior evidence. Never score below 50%. |
| Public roadmap | The "20% cap" rule prevents voting from hijacking the roadmap while preserving the trust benefit. "No public dates" is the strongest single-rule protection against sandbagging. |
| Integrations | Productlane is the only platform where the roadmap auto-mirrors your issue tracker (Linear). All others require manual status sync or webhook-level integration. |
