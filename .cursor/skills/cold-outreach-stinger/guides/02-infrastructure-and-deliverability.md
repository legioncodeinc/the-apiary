# Infrastructure and Deliverability

Email infrastructure is the foundation everything else runs on. Since Google's November 2025 enforcement change, getting this wrong means permanent 550 rejections, not temporary deferrals. Fix infrastructure before touching copy, sequences, or personalization.

## 2026 enforcement landscape

### Google (November 2025 changes)
- **Permanent 550 rejections:** As of November 2025, Google no longer issues temporary deferrals for authentication failures. Non-compliant senders receive permanent `550 5.7.26` errors. All bulk senders (1000+ emails/day to Gmail) must have SPF + DKIM + DMARC configured.
- **Google Postmaster Tools v2 (October 2025):** Reputation scoring changed from gradient (high/medium/low) to binary Pass/Fail. Check at postmaster.google.com.
- **Spam rate threshold:** Keep spam rate below 0.10% (hard limit is 0.30%, at which point delivery stops). Monitor daily in Postmaster.

### Microsoft (May 2025)
- Microsoft announced similar bulk sender requirements in May 2025. Specific requirements reference SPF + DKIM + DMARC.

## Non-negotiable setup checklist

Run through `templates/deliverability-audit-checklist.md` for every new setup. Key items:

### 1. Separate sending domains (blocking)
- Main domain (`company.com`) is NEVER used for cold outreach
- Create 2-3 dedicated cold outreach domains: `trycompany.com`, `getcompany.io`, `hellocompany.co`
- Set up forwarding from cold domains to main domain (emails look professional, replies arrive in your inbox)
- MX records point to your sending platform (Instantly or Smartlead)

### 2. DNS authentication (blocking)
- **SPF:** `v=spf1 include:[your-sending-platform.com] ~all` (one SPF record per domain, multiple includes as needed)
- **DKIM:** 2048-bit key, generated by your sending platform. Add the CNAME record to your DNS.
- **DMARC:** minimum `p=none` to start, monitor reports for 2 weeks, then move to `p=quarantine`. Do NOT jump straight to `p=reject` without reviewing reports first.
- Validate at MXToolbox: https://mxtoolbox.com/SuperTool.aspx

### 3. Mailbox setup (blocking)
- Use Google Workspace or Microsoft 365 mailboxes (not third-party ESP accounts for cold sending)
- 1-2 mailboxes per cold domain (more looks spammy)
- Complete name, profile photo, and professional signature on every mailbox
- No generic addresses (info@, sales@, contact@)

## Warmup protocol

Warmup is the process of building sending reputation for a new domain and mailbox before running cold campaigns. Skipping warmup produces immediate spam placement.

### Duration
- **Minimum:** 4 weeks for a fresh domain
- **Key metric during warmup:** reply rate (not send volume). Warmup tools simulate replies; monitor that warmup emails are receiving replies, not just sends.

### Volume ramp schedule
| Week | Sends/mailbox/day |
|---|---|
| 1 | 5-10 |
| 2 | 15-25 |
| 3 | 30-50 |
| 4 | 50-80 |
| Week 5+ (live campaigns) | 50-100 max |

Scale horizontally: add more mailboxes rather than increasing volume per mailbox.

### Warmup tools comparison (2026)
Based on `research/external/2026-05-20-email-warmup-tools-comparison.md`:
- **Smartlead built-in warmup:** 97% deliverability benchmark, included in Smartlead plans
- **Instantly warmup network:** included in Instantly plans, large warmup pool
- Both tools provide adequate warmup for standard use cases. No need for a separate warmup tool if already using Instantly or Smartlead.

### Warmup red flags
- Spam folder placement during warmup (most warmup tools track this)
- Warmup reply rate below 30% (suggests the warmup pool is low quality)
- DKIM validation failures in warmup network feedback

## Ongoing monitoring

After warmup, monitor these signals weekly:
- Google Postmaster spam rate (keep below 0.10%)
- Google Postmaster domain/IP reputation (must be Pass in v2)
- Bounce rate (keep below 3%)
- Spam complaint rate in sending platform dashboard

If spam rate exceeds 0.10%:
1. Pause campaigns immediately
2. Remove all non-verified contacts
3. Run deliverability diagnostic with `templates/deliverability-audit-checklist.md`
4. Do not resume until root cause is identified
