## Key Finding per Guide Area

### Platform Selection (`guides/01-platform-selection.md`)

**Most important finding:** The "6 platforms" brief is effectively 5 as of March 2026 (Drift sunset). The B2B Slack-native category (Pylon vs Plain) requires its own decision framework separate from widget-based platforms (Intercom, Crisp, Help Scout). The decision tree is: B2B Slack-heavy → Plain vs Pylon; PLG/B2C → Intercom; Email-first → Help Scout; Bootstrapped small team → Crisp.

**The strongest recommendation framework from research:**
- Developer-tool companies, API-first B2B: **Plain** ($35/seat, AI included, GraphQL, BYOA)
- Ops-led B2B, 50+ Slack channels, non-technical admins: **Pylon** ($59/seat, visual builder, account intelligence)
- VC-backed PLG SaaS, in-app messaging needed: **Intercom Early Stage** ($65/month year 1)
- Bootstrapped, small team, predictable cost: **Crisp Unlimited** (€95/month, unlimited agents)
- Email-first, simplicity valued: **Help Scout Standard** ($22/user/month)

### Widget Integration (`guides/02-widget-integration.md`)

**Most important finding:** The `ahize` open-source library (github.com/productdevbook/ahize, April 2026) provides a typed, zero-dependency abstraction over Intercom, Crisp, Zendesk, and Chatwoot's widget APIs with a unified `load/identify/track/show` interface. It may be worth referencing as a "multi-platform migration" option. For individual integrations, the canonical pattern is: (1) async-load the widget script, (2) fetch identity hash from a server-side API route, (3) boot/identify the widget with the hash. Next.js App Router considerations: use `use client` for the widget component, fetch identity from a Server Action or API Route.

---

### HMAC Identity Verification (`guides/03-identity-verification.md`)

**Most important finding:** The universal pattern is HMAC-SHA256 on a user identifier, signed server-side, hex-encoded, passed to the widget boot call. Intercom is migrating to JWT (recommend JWT for new implementations). Crisp uses email as the message (not user_id). The `Cache-Control: no-store` header on identity API responses is critical and often missed. Rotation procedure is platform-specific: Intercom has a dedicated Secrets API; Crisp requires generating a new secret in the dashboard.

---

### Conversation Routing (`guides/04-conversation-routing.md`)

**Most important finding:** The routing primitive taxonomy (inbox/team/tag/attribute/skills/load/intent/sentiment/overflow/priority/SLA) is consistent across platforms, but the implementation varies significantly. The highest-ROI routing configuration for SaaS is a priority queue for paying customers (reduces churn). The overflow-to-human rule is mandatory; no routing configuration should allow a conversation to queue indefinitely. Pylon's routing model is account-centric (map Slack channel to account to CRM owner), which is fundamentally different from conversation-centric routing in widget-based tools.

---

### AI Deflection (`guides/05-ai-deflection.md`)

**Most important finding:** Intercom Fin 2.0 achieves 51% resolution rate at $0.99/resolution. The key configuration principle (from official Intercom docs) is: sparse, data-driven Escalation Rules > fuzzy Escalation Guidance. The Recommendations engine (weekly AI gap analysis) should be used to prioritize knowledge base article creation rather than trying to pre-populate the knowledge base manually. Crisp Bot is less capable than Fin but included in the Unlimited plan (€95/month vs $0.99/resolution at Intercom). Plain's Ari AI is included on all plans with no per-resolution fees.

---

### Data Export (`guides/06-data-export.md`)

**Most important finding:** Intercom supports GDPR data export as CSV (via UI) and JSON (via API or Data Export → S3). GDPR Article 20 requires structured, machine-readable format. The discipline must be set up on day one because: (a) conversation history volume makes retrospective exports complex, (b) platform lock-in is permanent once customers expect support data to stay in a system. The stinger must include a "Day 1 Data Export Setup" checklist and a platform-by-platform export path matrix. Note: Drift customers migrating must export conversation data before the sunset completes.
