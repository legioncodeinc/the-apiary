# 05 — Quarterly Cadence: Cycle Anatomy, Check-in Ritual, CFR, Grading

The mechanics of running a healthy OKR cycle from kickoff through retrospective.

## The quarterly cycle anatomy

A well-run OKR cycle has five defined moments:

```
Week 1       Week 3-4      Week 7        Week 11       Week 12-13
   |             |            |             |               |
KICKOFF    BASELINE       MID-QTR       FINAL PUSH     END-OF-QTR
PLANNING    LOCK          CHECK-IN                    SCORING + RETRO
```

### 1. Kickoff (Week 1)

**Purpose:** Align on Objectives, finalize Key Results, establish baselines.

**Outputs:**
- Agreed Objectives (qualitative, aspirational, cycle-scoped)
- Key Results with: metric name, current baseline, target value, unit of measure, owner
- OKR type per KR: aspirational or committed (see `guides/04-calibration.md`)
- Check-in schedule for the quarter

**Common failure:** Rushing this. Teams that skip baseline-setting cannot grade their OKRs fairly at end of cycle. "Improve NPS" without a current NPS value is ungrадable.

### 2. Baseline lock (Week 3-4)

**Purpose:** Confirm baselines are measured, not estimated.

For any KR where the baseline was estimated at kickoff, teams have 2-4 weeks to pull real data. Lock the baseline before the mid-quarter check-in. Adjust the target if the actual baseline reveals the original target was set against a wrong assumption.

### 3. Mid-quarter check-in (Week 6-7)

**Purpose:** Confidence rating, blocker surfacing, trajectory assessment.

**Format (15-30 minutes per team):**
- Per KR: "On a 1-10 scale, how confident are we of hitting the target by end of quarter?"
- Confidence < 5: what's blocking it? Can it be unblocked?
- Confidence > 8 on an aspirational KR: was the target sandbagged?
- Flag any OKRs that have become irrelevant due to changed business context

**What this is NOT:** A progress presentation. It is a course-correction conversation.

### 4. Weekly status pulse (optional but recommended)

**Format:** One sentence per OKR. The "RAG status":
- **Green:** On track; no blocker.
- **Amber:** Behind; have a recovery plan.
- **Red:** Off track; need help or a decision.

This replaces status-update meeting time with asynchronous text. 15Five, Weekdone, and Lattice all support this natively.

### 5. End-of-quarter scoring + retrospective (Week 12-13)

**Purpose:** Honest scoring, learning extraction, next-cycle inputs.

**Scoring:**
- Use the 0.0–1.0 scale (see `guides/04-calibration.md` for score meaning)
- Score each KR independently before scoring the Objective
- Objective score = average of KR scores (or use judgment if KRs are unequally weighted)
- Do NOT retroactively adjust targets to make scores look better

**Retrospective question set (use any 3-5):**
1. What did we learn from this OKR cycle that we did not know at the start?
2. Which KRs were honest stretch targets, and which were sandbagged?
3. Which Objectives were truly aspirational, and which were operational tasks renamed?
4. What blockers appeared that we did not anticipate?
5. Did our OKRs reflect our actual priorities, or were our real priorities elsewhere?
6. What would we change about the process — cadence, format, tooling, ownership?
7. What KR baselines do we need to establish before next cycle begins?

## The CFR companion practice

CFR stands for Conversations, Feedback, Recognition — John Doerr's recommendation for the human infrastructure that makes OKRs work. Without CFR, OKRs become a quarterly paperwork ritual.

- **Conversations:** Regular 1:1s where managers and reports discuss OKR progress, not just status. "What does hitting this KR require that you don't have today?" is a CFR conversation. A status update is not.
- **Feedback:** Timely, specific, behavior-focused feedback tied to OKR work. Not annual performance review feedback.
- **Recognition:** Public acknowledgment of contributions toward Objectives. Recognition tied to process quality and learning, not just KR scores.

**Minimum viable CFR for small teams:** One 15-minute monthly conversation per person about their OKR progress, plus a public shout-out when someone clears a meaningful blocker or contributes an unexpected insight.

## Check-in anti-patterns

| Anti-pattern | Effect | Fix |
|---|---|---|
| Mid-quarter check-in becomes status theater | No actual course correction | Ask "what decision can we make today based on this data?" |
| Confidence ratings are always 7+ | Real blockers hidden | Ask "what would need to go wrong for this to be a 4?" |
| Scoring happens in 10 minutes at end of quarter | No learning extracted | Allocate 60-90 minutes; use the retrospective question set |
| OKR scores feed into performance reviews | Scoring stops being honest | Explicitly decouple OKR scores from compensation before the cycle starts |
| Objectives are never updated even when business context changes | OKRs become irrelevant | If a major business event changes priorities, it is legitimate to update Objectives mid-cycle; KR targets should be locked except in extraordinary circumstances |
