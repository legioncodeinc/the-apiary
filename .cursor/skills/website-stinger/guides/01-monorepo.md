# 01 — Monorepo Architecture

Source PRD: `research/source-prds/prd-phase-01-monorepo-architecture.md`

---

## Goal

Scaffold the pnpm workspace root containing `apps/web` (SvelteKit public site) and, in Payload mode, `apps/cms` (Next.js + Payload). Both apps deploy to Vercel as separate projects from the same repo.

---

## CMS-mode branch

| Mode | Apps scaffolded |
|---|---|
| Payload (default) | `apps/web` + `apps/cms` |
| TypeScript-as-CMS (fallback) | `apps/web` only |

---

## Scaffold steps

### 1. Workspace root

```bash
mkdir <project-name> && cd <project-name>
git init
```

Create `pnpm-workspace.yaml`:

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

Root `package.json`:

```json
{
  "name": "<project-name>-monorepo",
  "private": true,
  "scripts": {
    "dev:web": "pnpm --filter web dev",
    "dev:cms": "pnpm --filter cms dev",
    "build:web": "pnpm --filter web build",
    "build:cms": "pnpm --filter cms build"
  },
  "engines": { "node": ">=22.0.0", "pnpm": ">=9.0.0" }
}
```

### 2. SvelteKit app — apps/web

```bash
pnpm create svelte@latest apps/web
# Select: Skeleton project, TypeScript, ESLint, Prettier, Vitest
cd apps/web && pnpm install
pnpm add @sveltejs/adapter-vercel @sveltejs/enhanced-img
pnpm add -D tailwindcss @tailwindcss/vite
```

Key files:
- `apps/web/svelte.config.js` — use `adapter-vercel` (see `seo-aeo-stinger/templates/sveltekit/svelte.config.js`)
- `apps/web/vite.config.ts` — add `@tailwindcss/vite` and `enhancedImages()` plugins
- `apps/web/src/app.css` — import Tailwind v4 and design tokens
- `apps/web/src/app.html` — set `lang="en"`, viewport
- `apps/web/vercel.json`:

```json
{
  "framework": "sveltekit",
  "buildCommand": "pnpm build",
  "outputDirectory": ".svelte-kit/cloudflare",
  "installCommand": "pnpm install"
}
```

### 3. Payload CMS app — apps/cms (Payload mode only)

```bash
pnpm create payload-app@latest apps/cms
# Select: blank template, PostgreSQL (not MongoDB), TypeScript
cd apps/cms && pnpm install
```

Key files generated:
- `apps/cms/payload.config.ts` — Payload collections, Globals, db adapter, cors, CORS origins
- `apps/cms/src/app/(payload)/` — Payload admin routes (auto-generated by Payload)
- `apps/cms/src/app/api/[...slug]/route.ts` — Payload REST API catch-all

`apps/cms/vercel.json`:

```json
{
  "framework": "nextjs",
  "buildCommand": "pnpm build",
  "installCommand": "pnpm install"
}
```

### 4. Supabase CLI setup

```bash
# In monorepo root
pnpm add -D supabase
npx supabase init
```

`supabase/` folder at monorepo root holds migrations. Both apps share one Supabase project.

### 5. Shared types package (optional, Payload mode)

```bash
mkdir -p packages/payload-types
```

After running `payload generate:types` in `apps/cms`, copy or symlink `payload-types.ts` into `packages/payload-types/index.ts`. Add to both apps' `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@payload-types": ["../../packages/payload-types/index.ts"]
    }
  }
}
```

## Smoke check

```bash
# Terminal 1
cd apps/web && pnpm dev  # → http://localhost:5173

# Terminal 2 (Payload mode only)
cd apps/cms && pnpm dev  # → http://localhost:3000/admin
```

Both should load without errors.
