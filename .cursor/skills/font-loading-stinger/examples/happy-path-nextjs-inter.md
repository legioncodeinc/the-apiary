# Example: Next.js 15 + Inter Variable — Zero CLS Setup

**Scenario:** A Next.js 15 App Router project that uses Inter as the primary body font and wants zero CLS, correct preloading, and a properly typed font token system.

**Result:** Font payload ~35 kB, zero CLS, FOIT-free, ready for Tailwind v4.

---

## Step 1: Install nothing (next/font is built-in)

`next/font` is included in Next.js 13+. No installation needed.

## Step 2: Create `app/fonts.ts`

```typescript
// app/fonts.ts
import { Inter } from 'next/font/google';

export const inter = Inter({
  subsets: ['latin'],        // Download Latin subset only (~35 kB vs ~300 kB unsubsetted)
  display: 'optional',       // No CLS. First-load on slow connections may use system font.
  variable: '--font-inter',  // CSS custom property for Tailwind integration
  preload: true,             // Emit <link rel="preload"> with correct crossorigin
  fallback: [
    'system-ui',
    '-apple-system',
    'BlinkMacSystemFont',
    'Segoe UI',
    'sans-serif',
  ],
});
```

**Why `display: 'optional'`?** Inter is used for body copy. On first load with a cold cache, the system font renders for the content area. The visual difference between Inter and a well-matched system font (e.g., `system-ui` on macOS = San Francisco, visually close to Inter) is minimal. On repeat visits, Inter is cached and loads instantly. Zero CLS in exchange for a small visual difference on first cold load is the correct trade-off for body copy.

## Step 3: Apply in root layout

```typescript
// app/layout.tsx
import type { Metadata } from 'next';
import { inter } from './fonts';
import './globals.css';

export const metadata: Metadata = {
  title: 'My App',
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en" className={inter.variable}>
      {/* inter.variable adds --font-inter CSS custom property to <html> */}
      <body className={inter.className}>
        {/* inter.className directly applies the Inter font via generated class */}
        {children}
      </body>
    </html>
  );
}
```

## Step 4: Tailwind v4 integration

```css
/* app/globals.css */
@import 'tailwindcss';

@theme {
  --font-sans: var(--font-inter);
}
```

Now `font-sans` in Tailwind uses Inter. Components use `className="font-sans"` or it applies via the base layer automatically.

## Step 5: Verify in the browser

1. Open DevTools → **Network** tab → filter "Font"
2. Expected: one font request for Inter (WOFF2, ~35 kB, Priority: Highest)
3. Check that the font appears only ONCE (no double-fetch)
4. Check the request initiator is `<link rel="preload">` (not the CSS parser)

Run Lighthouse (Performance audit):
- CLS should be 0.0 from fonts
- "Preload key requests" should not flag Inter (it's already preloaded)
- "Font display" audit should pass (all fonts have `font-display`)

---

## What next/font generates (for verification)

In the browser, inspect the `<head>` generated by Next.js. You should see:

```html
<link
  rel="preload"
  as="font"
  href="/_next/static/media/inter-latin-variable.woff2"
  crossorigin="anonymous"
  type="font/woff2"
/>
```

And in the embedded `<style>`:

```css
@font-face {
  font-family: '__Inter_xxx';
  font-style: normal;
  font-weight: 100 900;
  font-display: optional;
  src: url('/_next/static/media/inter-latin-variable.woff2') format('woff2');
  unicode-range: U+0000-00FF, ...;
}

/* Metric-matched fallback (generated automatically by next/font) */
@font-face {
  font-family: '__Inter_Fallback_xxx';
  font-style: normal;
  font-weight: 100 900;
  font-display: optional;
  src: local('Arial');
  ascent-override: 90.20%;
  descent-override: 22.48%;
  line-gap-override: 0.00%;
  size-adjust: 107.40%;
}
```

The metric-matched fallback `@font-face` is generated automatically by `next/font`. Verify it contains `size-adjust` and the override values.
