# Reports

This folder collects past audit outputs produced by `slack-app-worker-bee`. Each report is a dated markdown file named `YYYY-MM-DD-<context>.md`.

A typical report includes:
- **Scenario classified:** new app scaffold / slash command addition / Block Kit audit / Events API setup / OAuth flow review / App Directory submission readiness.
- **Surfaces reviewed:** which Slack app surfaces were examined.
- **Findings:** numbered list of issues found, each with severity (Critical / High / Medium / Low), a description, and the relevant guide reference.
- **Recommendations:** concrete next steps for each finding.
- **Handoffs:** items routed to peer Bees (`devops-worker-bee`, `security-worker-bee`, `python-worker-bee`).

No reports yet. Reports accumulate here over time as `slack-app-worker-bee` completes sessions.
