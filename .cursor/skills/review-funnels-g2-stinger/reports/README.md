# Reports: review-funnels-g2-stinger

This folder collects past reputation-audit reports produced by `review-funnels-g2-worker-bee`.

## Report naming convention

`YYYY-MM-DD-<product-slug>-review-audit.md`

## Standard report shape

Each audit report includes:

1. **Executive summary** (2-3 sentences: current state, priority action)
2. **Platform presence scorecard** (G2, Capterra, Trustpilot, Product Hunt, AppSumo -- profile completeness, review count, average rating, badge status)
3. **Review velocity** (reviews per month, trend, benchmark against category average)
4. **Incentive compliance check** (current review-request program vs. G2 policy and FTC rule)
5. **Negative review triage** (open 1-2 star reviews with no response; priority responses needed)
6. **Badge status and opportunities** (which badges are earned, which are within reach, paid-plan ROI estimate)
7. **Recommended actions** (prioritized by impact)

## Frequency

Reputation audits are typically run:
- On-demand (user requests an audit)
- Quarterly (aligns with G2 badge award windows: Feb, May, Aug, Nov)
- After a Product Hunt launch
- After a significant spike in negative reviews
